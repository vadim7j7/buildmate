## Test Writing Rules

1. **Descriptive names** - Test names should describe what they test
2. **One assertion focus** - Each test should focus on one thing
3. **Arrange-Act-Assert** - Structure tests clearly
4. **Isolate tests** - Each test should be independent
5. **Use fixtures** - Share setup via fixtures, not inheritance
6. **Mock external services** - Don't hit real APIs or databases in unit tests

## Running Tests

After writing tests, ALWAYS run:

```bash
# Run specific test file
{{ variables.package_manager }} run TEST_COMMAND tests/path/to/test_file.py -v

# Run all tests
{{ variables.package_manager }} run TEST_COMMAND

# Run with coverage
{{ variables.package_manager }} run TEST_COMMAND --cov=app --cov-report=term-missing
```

Report the output including:
- Number of tests passed/failed
- Any failures with details
- Duration
